## 2) MySQL: Schema & Data Loading

**Q13: Table structure**

**Q14: Primary key**
ЁЯСЙ Use the unique ID column

**Q15: MySQL data types**
ЁЯСЙ Use `DECIMAL` for amount, `VARCHAR` for text, `INT` for counts

**Q20: Check truncation/import errors**

## 3) SQL Analytics

### Customer Behavior

**Q21: Highest total spend**
ЁЯСЙ Use `SUM()` + `GROUP BY AccountID`

**Q22: Most transactions**
ЁЯСЙ Use `COUNT(*)` + `GROUP BY AccountID`

**Q23: Average transaction per customer**
ЁЯСЙ Use `AVG(TransactionAmount)`

**Q24: Highest single transaction**
ЁЯСЙ Use `MAX(TransactionAmount)`

### Spending Trends

**Q25: Most frequent transaction type**
ЁЯСЙ Use `COUNT(*)` + `GROUP BY TransactionType`

**Q26: Transaction type with highest total value**
ЁЯСЙ Use `SUM(TransactionAmount)` + `GROUP BY TransactionType`

**Q27: Highest spending merchant categories or locations**
ЁЯСЙ Use `SUM(TransactionAmount)` + `GROUP BY Location`

**Q28: Spending by month**
ЁЯСЙ Use `DATE_FORMAT(TransactionDate, '%Y-%m')`

**Q29: Spending by day of week**
ЁЯСЙ Use `DAYNAME(TransactionDate)`

**Q30: Hour with most transactions**
ЁЯСЙ Use `HOUR(TransactionDate)`

### Risk / Suspicious Activity

**Q31: High value, late night, or high login attempts**
ЁЯСЙ Use `WHERE` with flag columns

**Q32: Locations with most suspicious transactions**
ЁЯСЙ Use `WHERE SuspiciousTransaction = 1`

**Q33: Customers most frequent in flagged transactions**
ЁЯСЙ Use `GROUP BY AccountID` with suspicious filter

**Q34: Channels with highest risk**
ЁЯСЙ Use `GROUP BY Channel` and count suspicious rows

**Q35: Suspicious volume vs total volume**
ЁЯСЙ Use `COUNT(CASE WHEN ...)`

### Customer Profile Insights

**Q36: Amount by age group**
ЁЯСЙ Use `CASE WHEN` for age bands

**Q37: Occupations that spend the most**
ЁЯСЙ Use `SUM(TransactionAmount)` + `GROUP BY CustomerOccupation`

**Q38: Age group with highest average transaction**
ЁЯСЙ Use `CASE WHEN` + `AVG()`

**Q39: Occupation with highest number of transactions**
ЁЯСЙ Use `COUNT(*)` + `GROUP BY CustomerOccupation`

**Q40: Account balance vs transaction amount**
ЁЯСЙ Use scatter chart or compare averages by balance bands
