## 2) MySQL: Schema & Data Loading

**Q13: Table structure**
👉 Use `CREATE TABLE`

**Q14: Primary key**
👉 Use the unique ID column

**Q15: MySQL data types**
👉 Use `DECIMAL` for amount, `VARCHAR` for text, `INT` for counts

**Q16: TransactionDate / PreviousTransactionDate type**

👉 Use `LOAD DATA INFILE`

**Q18: Verify all rows loaded**

**Q19: Check numeric columns imported correctly**

## 3) SQL Analytics

### Customer Behavior

**Q21: Highest total spend**
👉 Use `SUM()` + `GROUP BY AccountID`

**Q22: Most transactions**
👉 Use `COUNT(*)` + `GROUP BY AccountID`

**Q23: Average transaction per customer**
👉 Use `AVG(TransactionAmount)`

**Q24: Highest single transaction**
👉 Use `MAX(TransactionAmount)`

### Spending Trends

**Q25: Most frequent transaction type**
👉 Use `COUNT(*)` + `GROUP BY TransactionType`

**Q26: Transaction type with highest total value**
👉 Use `SUM(TransactionAmount)` + `GROUP BY TransactionType`

**Q27: Highest spending merchant categories or locations**
👉 Use `SUM(TransactionAmount)` + `GROUP BY Location`

**Q28: Spending by month**
👉 Use `DATE_FORMAT(TransactionDate, '%Y-%m')`

**Q29: Spending by day of week**
👉 Use `DAYNAME(TransactionDate)`

**Q30: Hour with most transactions**
👉 Use `HOUR(TransactionDate)`

### Risk / Suspicious Activity

**Q31: High value, late night, or high login attempts**
👉 Use `WHERE` with flag columns

**Q32: Locations with most suspicious transactions**
👉 Use `WHERE SuspiciousTransaction = 1`

**Q33: Customers most frequent in flagged transactions**
👉 Use `GROUP BY AccountID` with suspicious filter

**Q34: Channels with highest risk**
👉 Use `GROUP BY Channel` and count suspicious rows

**Q35: Suspicious volume vs total volume**
👉 Use `COUNT(CASE WHEN ...)`

### Customer Profile Insights

**Q36: Amount by age group**
👉 Use `CASE WHEN` for age bands

**Q37: Occupations that spend the most**
👉 Use `SUM(TransactionAmount)` + `GROUP BY CustomerOccupation`

**Q38: Age group with highest average transaction**
👉 Use `CASE WHEN` + `AVG()`

**Q39: Occupation with highest number of transactions**
👉 Use `COUNT(*)` + `GROUP BY CustomerOccupation`

**Q40: Account balance vs transaction amount**
👉 Use scatter chart or compare averages by balance bands
